# Specifications

Store project specifications here.

## Structure

```
specs/
└── [project-name]/
    ├── spec.md           # Main specification
    └── assets/           # Optional: diagrams, mockups
```

## Creating a New Spec

1. Create project folder: `mkdir specs/my-project`
2. Use the `spec-driven-development` skill to create `spec.md`
3. Or copy template from `.github/skills/spec-driven-development/templates/spec-template.md`

## Naming Conventions

- Use lowercase with hyphens: `my-awesome-project`
- Match the repository name if this is a dedicated repo
- Use feature names for feature specs: `user-authentication`
